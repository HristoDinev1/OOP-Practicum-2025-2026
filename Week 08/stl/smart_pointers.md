# Smart Pointers в C++

Умните указатели са обвивки около raw указатели, които автоматично управляват жизнения цикъл на обектите в heap-a. Те гарантират, че паметта ще бъде освободена веднага щом обектът вече не е нужен, предотвратявайки **memory leaks**.

```cpp
#include <memory>
using namespace std;
```

---

## Сравнение: Raw Pointers vs. Smart Pointers

| Характеристика | Raw Pointers (`T*`) | Smart Pointers (`unique`, `shared`) |
|----------------|---------------------|--------------------------------------|
| **Освобождаване** | Ръчно с `delete` | Автоматично (чрез деструктор) |
| **Безопасност** | Риск от dangling pointers | Намалява рисковете значително |
| **Изключения** | Не са exception-safe | Автоматично чистят при грешка |

---

## 1. `std::unique_ptr` — Единствен собственик

Това е най-бързият и ефективен умен указател. Той гарантира, че **само един** указател притежава обекта в даден момент. Не може да се копира, може само да се мести (`move`).

```cpp
// Създаване (Препоръчително чрез make_unique)
unique_ptr<int> p1 = make_unique<int>(10);

// unique_ptr<int> p2 = p1; // ГРЕШКА! Не може да се копира

// Прехвърляне на собственост
unique_ptr<int> p2 = move(p1); // p1 става null, p2 поема обекта

if (!p1) {
    cout << "p1 е празен";
}
```

---

## 2. `std::shared_ptr` — Споделена собственост

Използва **брояч на референциите** (reference counting). Един обект може да има много собственици. Когато последният `shared_ptr`, сочещ към обекта, бъде унищожен, паметта се освобождава.

```cpp
// Създаване (Препоръчително чрез make_shared)
shared_ptr<int> s1 = make_shared<int>(100);

{
    shared_ptr<int> s2 = s1; // Броячът става 2
    cout << s1.use_count();   // Изписва 2
} // s2 излиза от обхват, броячът става 1

cout << s1.use_count();       // Изписва 1
```

---

## 3. `std::weak_ptr` — Наблюдател

`weak_ptr` не притежава обекта и **не увеличава брояча** на `shared_ptr`. Използва се за решаване на проблема с **циклични зависимости** (когато два обекта сочат един към друг и никога не се изтриват).

```cpp
shared_ptr<int> shared = make_shared<int>(42);
weak_ptr<int> weak = shared; // Не увеличава use_count()

// Трябва да го "заключим", за да го използваме
if (shared_ptr<int> temp = weak.lock()) {
    cout << *temp; // Обектът все още съществува
} else {
    cout << "Обектът е вече изтрит";
}
```

## Защо са "по-добри"? (Пример)

### Старият начин (Raw):
```cpp
void riskyFunction() {
    int* ptr = new int(10);
    if (someError) return; // ПАМЕТТА ИЗТИЧА ТУК (delete не се вика)
    delete ptr;
}
```

### Модерният начин (Smart):
```cpp
void safeFunction() {
    auto ptr = make_unique<int>(10);
    if (someError) return; // ПАМЕТТА СЕ ЧИСТИ АВТОМАТИЧНО при излизане от обхват
}
```

## Бърза таблица за избор

| Тип                | Кога да го използвам?                                      |
|--------------------|------------------------------------------------------------|
| `unique_ptr`       | По подразбиране. Когато обектът има само един owner.    |
| `shared_ptr`       | Когато обектът има няколко owners. |
| `weak_ptr`         | За избягване на цикли между `shared_ptr`.  |
| `make_unique/shared`| Винаги използвай тези функции вместо `new` за сигурност.   |
